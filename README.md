# 1. Сегмент + ценностное предложение
Целевой сегмент:
Фрилансеры, фотографы и видеографы (мобильные креаторы контента), работающие в полевых условиях (на выезде, в поездках), нуждающиеся в быстром доступе к большим объемам данных без кабелей.

Операционализированное ценностное предложение:
Для мобильных креаторов в ситуации необходимости быстрого доступа к файлам на съёмках или в поездках (без ноутбука под рукой) портативное SSD-хранилище обеспечивает беспроводную передачу данных на смартфон или ПК со скоростью до 100 МБ/с, улучшая время доступа к файлам на 70% по сравнению с USB-кабелями или облачными сервисами (которые требуют интернета), при условиях наличия Wi-Fi/Bluetooth на устройствах и расстояния до 10 метров.

Основания доверия ("почему пользователь поверит"):

SSD с емкостью 1–2 ТБ и скоростью чтения/записи 500+ МБ/с (стандарт для современных NVMe SSD).

Беспроводная передача через встроенный Wi-Fi 6 модуль (реальные аналоги: WD Wireless Pro).

Батарея на 8+ часов работы + компактный размер (как powerbank), протестировано на реальных сценариях съёмок.

# 2. Основной сценарий (user flow)
Основной сценарий: Первая беспроводная передача файлов с SSD на смартфон.

Триггер: Пользователь на съёмках записал видео на камеру и подключает её к SSD по USB для копирования.

Вход: Активирует SSD кнопкой питания (Wi-Fi сеть SSD появляется автоматически).

Подключение: На смартфоне в приложении сканирует и подключается к Wi-Fi SSD (пароль по умолчанию или QR-код).

Просмотр: В приложении видит список файлов на SSD, выбирает нужные (фильтр по типу/дате).

Передача: Запускает передачу выбранных файлов (прогресс-бар с ETA).

Получение результата: Файлы скачиваются на смартфон, подтверждает успешность (checksum).

Фиксация: SSD переходит в спящий режим; пользователь редактирует файлы в приложении (предпросмотр).

(Всего 7 шагов.)

# 3. Альтернативный сценарий
Альтернативный сценарий: Повторное использование с ошибкой подключения.

Триггер: Пользователь возвращается к SSD после перерыва (батарея разряжена частично).

Вход: Включает SSD; смартфон не видит сеть (ошибка: помехи Wi-Fi).

Диагностика: Приложение предлагает перезагрузку SSD или ручной ввод IP.

Подключение: Переподключается вручную или через Bluetooth fallback.

Передача: Выбирает файлы для обновления (синхронизация изменений).

Завершение: Подтверждает передачу; приложение логирует ошибку для отчёта.

(Всего 6 шагов.)

# 4. Карта функций (Core/Support/Later) — подробное описание

## Core (ядро MVP, 6 функций)


### Автоматическая Wi-Fi точка доступа

Описание:  SSD создаёт защищённую Wi-Fi сеть (SSID: "PortableSSD_XXXX") при включении. Поддержка Wi-Fi 6 для скоростей до 100 МБ/с.

Реализация: Чип ESP32 или аналог (с антенной 2.4/5 ГГц); автостартует AP в firmware.

Связь со сценарием: Шаг 3 основного и альтернативного (подключение).

Элемент ценности: Основной канал передачи (70% ускорение vs USB).


### Приложение для просмотра/скачивания файлов

Описание:  Мобильное app (iOS/Android) с файловым менеджером: список, превью, множественный выбор, скачивание.

Реализация: Flutter/Dart для кросс-платформы; WebDAV/SMB протокол для доступа к SSD.

Связь со сценарием: Шаги 4–5 основного.

Элемент ценности: Быстрый доступ и выбор файлов на ходу.


### Прогресс-бар и checksum

Описание: Визуальный индикатор передачи (процент, скорость, ETA); автоматическая проверка MD5/SHA-256 хэшей.

Реализация: В app — progress listener; на SSD — файловая утилита в firmware.

Связь со сценарием: Шаг 6 основного.

Элемент ценности: Надёжность (измеримый эффект без потерь данных).


### Включение/выключение

Описание: Физическая кнопка с LED-индикаторами (заряд, статус Wi-Fi, передача).

Реализация: Микроконтроллер GPIO + кнопка; soft power-off через таймер.

Связь со сценарием: Шаги 2 и 1 альтернативного.

Элемент ценности: Простота запуска (портативность).


### Браузерный доступ через IP

Описание: Веб-интерфейс (http://192.168.4.1) с тем же функционалом: просмотр, скачивание.

Реализация: Локальный веб-сервер на ESP32 (HTML/JS, no framework).

Связь со сценарием: Шаг 3 альтернативного (для ПК).

Элемент ценности: Универсальность без установки.


### Авто-спящий режим

Описание: Отключение Wi-Fi/дисковода при неактивности; пробуждение по кнопке.

Реализация: Таймер в firmware + power management чип.

Связь со сценарием: Шаг 7 основного.

Элемент ценности: 8+ часов автономии.

## Support (поддерживающие, 5 функций)
Улучшают удобство, но не критичны для базовой ценности.


### Bluetooth fallback

Описание: BT 5.0 для начального pairing и низкоскоростной передачи (до 10 МБ/с) при Wi-Fi сбое.

Реализация: ESP32 BT stack; app BLE сервис.


### QR-код для пароля

Описание: На экране app или наклейке — QR со SSID+паролем для 1-клик подключения.

Реализация: Генерация QR в app; камера для сканирования.


### Фильтры файлов

Описание: По типу (JPG, MP4), размеру (>100 МБ), дате; поиск по имени.

Реализация: JS-фильтр в app/веб.


### Предпросмотр

Описание: Тумбнейлы для фото/видео (первые 5 сек).

Реализация: FFmpeg-lite в app.


### Статистика использования

Описание: Локальные логи: объём переданного, время сессии.

Реализация: JSON-файл на SSD.

## Later (отложенные, 7 функций)
Для будущих версий, не влияют на MVP.


### Локальное шифрование

Описание: AES-256 для папок; unlock по PIN.


### Синхронизация изменений

Описание: Двусторонняя sync с телефоном.


### Мультиподключение

Описание: До 3 клиентов одновременно.


### Firmware update OTA

Описание: Загрузка обновлений по Wi-Fi.


### Ручной ввод IP

Описание: Для сетевых проблем.


### Логи ошибок

Описание: Экспорт отчётов.


### Расширенные фильтры

Описание: По метаданным (EXIF, GPS).

### 1	Как мобильный креатор, я хочу включить SSD кнопкой, чтобы быстро активировать беспроводной доступ.	

- Кнопка зажигает индикатор в <3 сек.
- Wi-Fi сеть появляется в <10 сек.
- Автоотключение через 5 мин без активности.
- Зависит от аппаратной кнопки.

### 2	Как пользователь, я хочу подключиться к Wi-Fi SSD через приложение, чтобы просматривать файлы.	

- Сканирование сетей в приложении.
- Подключение с паролем/QR за <15 сек.
- Отображение файловой структуры.
- Риск помех Wi-Fi.

### 3	Как креатор, я хочу выбрать и скачать файлы на смартфон, чтобы работать без кабелей.	

- Выбор нескольких файлов.
- Скорость >50 МБ/с.
- Прогресс-бар с ETA.
- Зависит от Wi-Fi чипа.

### 4	Как пользователь, я хочу проверить целостность скачанных файлов, чтобы избежать ошибок.	

- Авто-checksum после передачи.
- Уведомление об ошибках.
- Передача повторяется автоматически.
- Нет.

### 5	Как пользователь ПК, я хочу доступ через браузер по IP, чтобы передавать на компьютер.	

- Доступ по http://192.168.x.x.
- Тот же интерфейс просмотра/скачивания.
- Работает без установки apps.
- Кросс-браузерность.

### 6	Как креатор, я хочу авто-спящий режим, чтобы SSD работал 8+ часов на батарее.	

- Спящий режим после 5 мин.
- Батарея >80% после 1 часа передачи.
- Индикатор заряда в приложении.
- Тестирование батареи.

### 7	Как повторный пользователь, я хочу Bluetooth fallback при Wi-Fi ошибке, чтобы не потерять доступ.	

- Авто-переход при неудаче Wi-Fi.
- Подключение через BT в <20 сек.
- Ограниченная передача (до 10 МБ/с).
- Доп. чип BT увеличивает стоимость.

# 6. Нефункциональные требования
Производительность: Передача файлов — скорость >50 МБ/с на Wi-Fi 6; отклик интерфейса <1 сек.

Доступность: Устройство весит <200 г, размер как powerbank (10x6x2 см); батарея 8000 мАч, 8+ часов.

Безопасность: WPA3 для Wi-Fi; опциональное PIN для доступа; данные шифруются на SSD (AES-256).

Надёжность: MTBF >50 000 часов; защита от падений 1 м; водозащита IP54.

Совместимость: iOS 14+, Android 10+, Windows/Mac (браузер Chrome/Edge); SSD NVMe PCIe 3.0+.

Масштабируемость: Поддержка до 4 ТБ SSD; firmware обновляемое OTA.

Локализация данных: Все данные хранятся локально, без облака; нет передачи на серверы.

Удобство интерфейса: Приложение на русском/английском; интуитивный UI (Material Design).

Все требования согласованы: шаги сценариев покрыты Core-функциями, нефункциональные не выходят за рамки MVP (бюджет ~$100/устройство на прототип). Риски минимальны при использовании ESP32 + NVMe-модуль.
